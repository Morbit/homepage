БЭМ

Если у каждой части сайта есть название, этими терминами удобно пользоваться при общении внутри команды.

  • Хотелки менеджера
  • Изменения требований
  • Дальнейший support
  • Добавление, перемещение, изменение

Блок

  • некая самостоятельная сущность
  • простые и составные (содержат в себе другие блоки)

Элемент

  • часть блока, отвечающая за отдельную функцию.
  • может находиться только в составе блока и не имеет смысла в отрыве от него.

Что нам надо описать?

  • Факт присутствия блоков
  • Порядок и расположение
  • Не забываем про вложенность блоков друг в друга

Как-то так

Имя блока и имя элемента

  • Имя блока уникально
  • Одинаковые имена только у одинаковых блоков
  • Имя элемента уникально в пределах блока
  • Одинаковых элементов в блоке может быть несколько
    		block: 'head',
    		content: [
    		  {
    		    block: 'search',
    		    content: [
    		      { elem: 'input' },
    		      { elem: 'button', content: 'Search' }
    		    ]
    		  }
    		]

БЭМ-дерево

  • JSON + BEMHTML = HTML

Независимые блоки

Независимый блок реализован так, чтобы его можно было вставить в любое место на странице. В том числе и вставить в другой блок.

  • У блока (или элемента) должно быть уникальное «имя»
  • Не должно быть CSS-селекторов на теги (.menu td)
  • Нужно избегать каскадные селекторы (.header .menu)

БЭМ-нотация

      	<ul class="menu">
      	  <li class="menu__item">…</li>
      	  <li class="menu__item">…</li>
      	</ul>

Повторяемость блоков

  • Может появиться второй блок
  • В любой момент может появиться второй такой же
  • Нельзя id
  • Нельзя tags
  • Всё дерево покрываем классами (без фанатизма, чувство меры)

Модифицируемость

Модификатор

это свойство блока или элемента, которое меняет внешний вид или поведение.

Модификаторы в БЭМ-дереве

				{
				  block: 'menu',
				  mods: [
				    { size: 'big' },
				    { type: 'buttons' }
				  ]
				}

Модификаторы блока

				<ul class="menu menu_size_big menu_type_buttons">
				</ul>
				 
				.menu_size_big {
				  // CSS code to specify height
				}
				.menu_type_buttons .menu__item {
				  // CSS code to change item's look
				}

Модификаторы элемента

				<ul class="menu">
				  <li class="menu__item">Index</li>
				  <li class="menu__item menu__item_state_current">…</li>
				  <li class="menu__item">Contact</li>
				</ul>

Немного о CSS

				.menu {
				  __item {
				    _state_current {
				      font-weight: bold;
				    }
				  }
				}

Немного о JS

				$('menu__item').click( … );
				$('menu__item').addClass('menu__item_state_current');
				$('menu').toggle('menu_size_big').toggle('menu_size_small');
				

				block('menu').elem('item').click( … );
				block('menu').elem('item').setMod('state', 'current');
				block('menu').toggleMod('size', 'big', 'small');

Консистентность блока

  • Блок должен знать о себе всё
  • Реализация блока подразумевает описание его внешнего вида и поведения во всех необходимых технологиях.
  • Мультилингвальность

Мультилингвальность

    • шаблоны, превращающие декларацию блока на странице в HTML-код
    • CSS, отвечающий за внешний вид блока
    • JavaScript-реализация блока, если у блока также есть динамическое поведение
    • Картинки, относящиеся к блоку
    • Документация к блоку
      	menu/
      	  __item/
      	    _state/
      	      menu__item_state_current.css
      	      menu__item_state_current.xsl
      	    menu__item.css
      	    menu__item.xsl
      	  menu.css
      	  menu.js
      	  menu.xsl
