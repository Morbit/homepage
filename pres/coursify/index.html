Проблема:

  • Не в том, что во всех браузерах всё выглядит по-разному
  • В том, что мы не хотим принимать их и их возможности AS IS и стремимся всё запилить пиксель в пиксель
  • Не надо заставлять летать рождённого ползать

Значит

  1. Пользователь должен получать контент в любом случае
  2. Дизайн не должен ему в этом мешать
  3. Реализация дизайна должна учитывать возможности конечного пользователя

Побеждаем многообразие

Graceful Degradation

Пилим сайт для самого крутого браузера и проверяем на доступность в других

  • Работаем в Хроме
  • Поправили фичу в FF
  • Открыли ИЕ, нет белых букв на белом фоне etc. OK.
  • Открыли в iPhone, вся информация доступна? ОК.

Progressive Enchancement

Начинаем с содержимого и улучшаем если позволяют возможности

  • HTML
  • простой CSS и простой JS
  • крутой CSS и крутой JS
  • офигенские селекторы CSS3, фичи HTML5 etc.

Что же лучше?

Mobile First, Web Second

Сначала проектируем мобильную версию

  • небольшое разрешение, одна колонка
  • удаляем второстепенные блоки
  • упор на самое важное

PE + MF = RWD

Responsive / Adaptive Web Design

Цель: универсальность сайта для различных устройств без создания отдельных версий

Зачем:
  • куча разрешений
  • популярность мобильных устройств
  • рост мобильного трафика

RWD пример


Особенности

  • Grid-based layout
  • Flexible images
  • MediaQueries

Flexible images 

				img {
				  max-width: 100%;
				}

MediaQueries

				@media screen and (max-width: 980px) {
				  // selectors for screens less than 980px width
				}

Популярные

				/* Large desktop */
				@media (min-width: 1200px) { ... }
				/* Portrait tablet to landscape and desktop */
				@media (min-width: 768px) and (max-width: 979px) { ... }
				/* Landscape phone to portrait tablet */
				@media (max-width: 767px) { ... }
				/* Landscape phones and down */
				@media (max-width: 480px) { ... }

Media features

  • device-width / device-height
  • orientation
  • device-aspect-ratio
  • monochrome
  • resolution
  • etc.
